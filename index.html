Este front envia os dados no formato exato esperado pelo fluxo 1 do n8n. Ele já monta o payload compatível com a criação do cliente no banco e no Asaas.

Antes de testar:
1. No n8n, use a URL de teste se o workflow estiver em execução manual, ou a URL de produção se ele estiver ativo.
2. No Webhook node, deixe Allowed Origins (CORS) como * ou inclua o domínio do seu HTML.
3. O fluxo 1 exige também dados de pagamento, então este HTML já envia esses campos junto.
Ex.: teste: /webhook-test/... | produção: /webhook/...
